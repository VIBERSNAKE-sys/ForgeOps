# ResForge Tier 2 Backlog — Port When Bandwidth Allows

These prevent specific failure modes that are serious but infrequent.
Not yet ported from the source ResForge modules.

## Items

1. **Zero-MUST Fallback:** When all JD requirements are LIKELY_MUST with 0 MUST, substitute LIKELY_MUST into the gap tally at face value. Same thresholds.

2. **No Cross-Entry Inference:** Never combine multiple article-digest entries to create a single claim.

3. **Formality Escalation Prohibition:** "briefed," "presented to leadership," "influenced decisions," "advised executives" require artifact evidence.

4. **Exposure-to-Evidence Promotion Lock:** Exposure-class language cannot be promoted to Evidence-class through inference.

5. **Full 8-rung Framing Ladder:** Expand from 3 tiers (DELIVER/COORDINATE/CONTRIBUTE) to 8 (OWN/DELIVER/COORDINATE/IMPLEMENT/ANALYZE/CONTRIBUTE/PARTICIPATE/FAMILIAR).

6. **Scan Quality Filter (Layoffs + Posting Patterns):** During batch scanning, run lightweight company health checks — Signal 1 (recent layoffs via WebSearch) and Signal 3 (posting age/ghost indicators). Filter out dead postings before they reach the evaluation queue. Requires WebSearch calls during scan, roughly doubles scan time.

## What NOT to port

- Checkpoints ("type continue") — career-ops's speed advantage requires single-pass generation
- Formal Evidence Ledger document for cold applications — two-track strategy reserves that for ResForge
- PAR Audit or Step 2F Project Compression — ResForge-only craft-level optimizations
